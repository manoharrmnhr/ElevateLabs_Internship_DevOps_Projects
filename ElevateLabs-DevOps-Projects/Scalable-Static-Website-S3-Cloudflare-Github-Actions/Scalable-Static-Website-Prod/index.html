System Design

Architecture Overview

A zero-server, globally distributed delivery pipeline with automatic failover and DDoS protection.

01
🪣

AWS S3 Origin

Static files stored in an S3 bucket with website hosting enabled. Bucket policy grants public read. S3 versioning enabled for instant rollback.

  • Free Tier — 5GB storage
  • Versioning & lifecycle rules
  • Region: ap-south-1 (Mumbai)
02
☁️

Cloudflare CDN

All traffic proxied through Cloudflare's global network. HTTPS termination at the edge, zero-cost DDoS protection, smart caching rules.

  • SSL/TLS — Flexible mode
  • Always Use HTTPS
  • Cache-Everything page rule
03
⚙️

GitHub Actions CI/CD

Triggered on every push to main. Two-job pipeline: validate HTML integrity, then sync to S3 with per-type cache headers and purge CDN.

  • Concurrent deploy guard
  • Environment secrets
  • HTTP smoke test post-deploy
CI/CD Flow

Deployment Pipeline

Every git push triggers a fully automated 6-step deployment pipeline.

1

Code Push

Developer pushes to main branch. GitHub receives the webhook event and queues the Actions workflow.

2

HTML Validation

Validate job checks all HTML files for DOCTYPE declaration and structural integrity before proceeding.

3

AWS Credential Config

Workflow authenticates using scoped IAM credentials stored as GitHub repository secrets.

4

Typed S3 Sync

HTML synced with no-cache headers. CSS/JS synced with 1-year immutable cache. Assets synced with 30-day cache.

5

Cloudflare Cache Purge

Entire CDN cache purged via Cloudflare API so all edge nodes serve fresh content immediately.

6

Smoke Test

Live URL fetched to verify HTTP 200 response. Deploy summary written to GitHub Actions job summary.
